Privacy Policy

Effective: April 8, 2026 · Last updated: April 12, 2026

Overview

LinkedComment ("the Extension", "we", "our") is a Chrome browser extension that detects trending posts on LinkedIn and X.com (Twitter) and generates AI-powered comments and replies. This privacy policy comprehensively describes how the Extension collects, uses, stores, and shares user data. By installing and using LinkedComment, you agree to the practices described in this policy.

1. Data Collection

The following is a complete list of all user data that LinkedComment collects or accesses:

a) Website Content (Post Text from LinkedIn and X.com)

When you click "Generate Comment" or "Generate Reply" on a post, the Extension reads the text content of that specific post from the active LinkedIn or X.com tab. This data is collected temporarily in your browser's memory solely for the purpose of generating a comment or reply.

b) User-Provided Settings

The Extension collects and stores the following settings that you explicitly provide through the Extension popup:

  • AI provider selection (Ollama, OpenAI, Google Gemini, or Anthropic Claude)
  • AI model name selection
  • API keys for your chosen AI provider (entered by you)
  • Ollama server URL (if using Ollama)
  • Trending post detection thresholds (reactions, comments, reposts numbers)
  • Dark mode preference
  • Trending post detector on/off toggle

c) Generated Comments

AI-generated comments are held temporarily in browser memory while displayed in the Extension widget. They are not persisted after the widget is closed.

d) Data NOT Collected

LinkedComment does not collect:

  • Personally identifiable information (name, email, address, etc.)
  • LinkedIn account credentials or login information
  • Browsing history or activity outside of LinkedIn
  • Analytics, telemetry, or usage statistics
  • Cookies or tracking identifiers
  • Location data
  • Financial or payment information
  • Health information
  • Communications or messages

2. How Data Is Used

Each type of collected data is used exclusively for the following purposes:

  • LinkedIn post text: Sent to your configured AI provider as part of a prompt to generate a relevant comment. Used only at the moment you click "Generate Comment" and not retained afterward.
  • User settings (provider, model, thresholds, dark mode): Used to configure the Extension's behavior according to your preferences. Read each time the Extension loads or you interact with the widget.
  • API keys: Used solely to authenticate requests to the AI provider you have selected. Sent only to the specific AI provider's API endpoint during comment generation.
  • Generated comments: Displayed to you in the widget. Available for you to copy to clipboard or insert into LinkedIn's comment box. Not used for any other purpose.

We do not use any collected data for advertising, marketing, analytics, profiling, or any purpose other than the core functionality described above.

3. Data Storage

a) Local Storage Only

All user settings and API keys are stored exclusively in Chrome's local extension storage (chrome.storage.local) on your device. This storage is:

  • Accessible only to the LinkedComment extension
  • Stored locally on your computer
  • Not synced to any cloud service or external server
  • Removed when you uninstall the Extension

b) No External Storage

LinkedComment does not operate any servers, databases, or cloud storage. We have no backend infrastructure. No user data is stored outside of your local browser environment.

c) Temporary Memory

LinkedIn post text and generated comments are held in your browser's working memory only during active use. This data is automatically cleared when you close the widget or navigate away from the page. It is never written to disk or persistent storage.

d) Data Retention

User settings and API keys persist in Chrome's local storage until you either change them, clear extension data, or uninstall the Extension. Temporary data (post text, generated comments) is not retained beyond the active browser session.

4. Data Sharing

The following is a complete list of all third parties that user data may be shared with, and what data is shared:

a) AI Service Providers (only when you generate a comment)

When you click "Generate Comment", the text of the LinkedIn or X.com post is sent to the AI provider you have configured. The specific provider depends entirely on your choice in the Extension settings:

  • Ollama (local): If you select Ollama, post text is sent to a server running on your own computer (localhost). No data leaves your device.
  • OpenAI (api.openai.com): If you select OpenAI, post text and your API key are sent to OpenAI's API servers. OpenAI processes this data according to their Privacy Policy and Terms of Use.
  • Google Gemini (generativelanguage.googleapis.com): If you select Gemini, post text and your API key are sent to Google's Generative AI API servers. Google processes this data according to their Gemini API Terms of Service and Privacy Policy.
  • Anthropic Claude (api.anthropic.com): If you select Claude, post text and your API key are sent to Anthropic's API servers. Anthropic processes this data according to their Privacy Policy and Terms of Service.

Important: Data is sent to only one provider at a time — the one you have actively selected. You control which provider receives your data. Communication occurs directly between your browser and the provider's API. LinkedComment does not route, intercept, log, or store any data in transit.

b) No Other Third Parties

Beyond the AI provider you explicitly configure, LinkedComment does not share, sell, rent, trade, or transfer user data to any other third party, including:

  • No advertising networks
  • No analytics services
  • No data brokers
  • No social media platforms (other than LinkedIn via your direct interaction)
  • No affiliated or parent companies
  • No government entities (unless required by law)

c) We Do Not Sell User Data

We do not sell, license, or monetize user data in any form.

5. Permissions Explained

The Extension requires the following Chrome permissions, each justified below:

  • activeTab: Allows the Extension to read content on the current LinkedIn or X.com tab when you interact with it. Used to extract post text for comment or reply generation. Only active on linkedin.com and x.com pages.
  • clipboardWrite: Allows the "Copy" button to write the generated comment to your system clipboard.
  • clipboardRead: Used alongside clipboardWrite to support clipboard operations for generated comments.
  • storage: Allows the Extension to save your settings (AI provider, model, thresholds, API keys) locally using Chrome's extension storage API.
  • declarativeNetRequest: Used to modify HTTP request headers (removes the Origin header) for requests to localhost and 127.0.0.1 only. This is required for communication with local Ollama AI servers, which enforce CORS restrictions. No external traffic is modified.
  • Host permissions — localhost/127.0.0.1: Required to send AI generation requests to a local Ollama server running on your machine.
  • Host permissions — api.openai.com: Required to send AI generation requests to OpenAI's API when you select OpenAI as your provider.
  • Host permissions — generativelanguage.googleapis.com: Required to send AI generation requests to Google's Gemini API when you select Gemini as your provider.
  • Host permissions — api.anthropic.com: Required to send AI generation requests to Anthropic's Claude API when you select Claude as your provider.

6. User Controls and Data Deletion

You have full control over your data:

  • Change or remove API keys: Open the Extension popup, clear the API key field, and click Save Settings.
  • Switch providers: Change your AI provider at any time in the Extension popup. Only the active provider receives data.
  • Delete all data: Uninstalling the Extension from Chrome automatically removes all stored settings and API keys from your device.
  • Clear extension data manually: Go to chrome://extensions, click "Details" on LinkedComment, then "Clear data" to remove all stored settings without uninstalling.

7. Security

API keys are stored in Chrome's local extension storage, which is sandboxed and accessible only to the LinkedComment extension. All communication with cloud AI providers occurs over HTTPS (TLS-encrypted connections). We do not transmit API keys or user data over unencrypted connections. However, no method of electronic storage is 100% secure, and we cannot guarantee absolute security.

8. Analytics, Tracking & Advertising

LinkedComment contains zero analytics, tracking, telemetry, crash reporting, advertising, or third-party scripts of any kind. We do not track your browsing activity, extension usage patterns, LinkedIn engagement metrics, or any other behavioral data.

9. Children's Privacy

LinkedComment is not directed at children under the age of 13. We do not knowingly collect any personal information from children under 13. If you believe a child under 13 has provided data through the Extension, please contact us so we can address the concern.

10. International Users

If you use a cloud AI provider (OpenAI, Google Gemini, or Anthropic Claude), the post text you send for comment or reply generation may be processed on servers located outside your country of residence. Please refer to each provider's privacy policy (linked in Section 4) for details about their data processing locations.

11. Changes to This Policy

We may update this privacy policy from time to time. When we make changes, we will update the "Last updated" date at the top of this policy. If we make material changes to how we handle user data, we will update the Extension description on the Chrome Web Store. We encourage you to review this policy periodically. Your continued use of the Extension after changes are posted constitutes acceptance of the updated policy.
