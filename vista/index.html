Why it works — and why nothing can detect it

Every NSWindow on macOS has a property called sharingType. Set it to .none and the window becomes a black hole to anything trying to capture pixels. It's been in Apple's SDK since Mavericks (2013) — almost nobody uses it.

There's no kernel extension, no hook, nothing for software to detect. The operating system itself removes the window from the capture stream before any app — Zoom, Meet, Teams, proctoring tools — ever sees it. It can't “stop working” after a macOS update, because it's a standard, documented Apple API working exactly as designed.

Note: Vista hides its window from the digital screen-capture pipeline only. A physical camera pointed at your monitor still sees whatever is on the monitor — this is a software solution to a software problem.

FAQ

Good questions, honest answers

Does this hide the browser from a camera pointing at my screen?

No. Vista hides its window from the digital screen-capture pipeline. A camera looking at your monitor will still see whatever is on the monitor. This is a software solution to a software problem.

Will it be detected by Zoom / Meet / Teams / proctoring software?

Vista uses a standard, documented Apple API (NSWindow.sharingType) that the operating system honors. There's no kernel extension and no hook — because the OS itself removes the window from the capture stream before any app sees it.

Does it work on Windows or Linux?

No. This is a macOS-specific feature using a macOS-specific API. macOS only.
